# 10. v3 跨网部署与运维（Tailscale 直连 + git 数据面）

> 定位：把 [09 规格](09-v3协作系统规格.md) §7（网络/鉴权）与 §2.6（控制面/数据面分离）的 **what**，落成可执行的 **how**。面向"想让两台以上机器、跨网协作"的运维者。代码连接层（`abg broker start` 绑可配 host、`AGENTBRIDGE_BROKER_URL` 连远端、PSK 握手）已就绪，本篇是接它的操作手册。
>
> **会话集成范围（诚实标注）**：v3 房间事件「自动推进活 session + 完成自动发布」目前**仅 Claude Code** 端到端打通（见 [09 §5.5](09-v3协作系统规格.md)）。控制面 agent-无关——任何持 PSK token 的进程都能当房间成员收发事件；但本手册里「agent 自动获知/自动发布」的体验当前以 **Claude Code 会话**为准。Codex 等其它 agent 的 v3 房间会话集成是明确 backlog（[09 §10.1](09-v3协作系统规格.md)）。

---

## 1. 部署形态选型

| 形态 | 何时用 | 加密 |
|------|--------|------|
| **内网常开机 + Tailscale**（推荐默认） | 团队各自机器/家里小主机/公司内网，想零公网暴露 | WireGuard 链路自带加密，broker 用明文 `ws://100.x:4700` 即可 |
| 公网 VPS | 必须公网可达、无法都上同一 tailnet | broker **无内置 TLS**，必须前置反代（Caddy/nginx）终结 `wss://`，再回源到 broker |

绝大多数团队选第一种。下文以 Tailscale 形态为主。

---

## 2. 双层鉴权总览

两层叠加，缺一不可：

1. **网络层（Tailscale ACL）**：设备没打 `tag:agent` → 连 `100.x:4700` 直接被 tailnet ACL 拒，WS 都建立不起来。
2. **应用层（broker PSK）**：已在 tailnet 内的设备，仍需 `abg auth login` 签发的 token；握手失败 broker 直接 4401 关连接，且从不回显 token。

> 🔸 **重要边界：鉴权 ≠ room 授权**。当前 MVP 是「单团队、共享 tailnet、扁平信任」模型——broker 的 `subscribe` 不做 room 成员校验，**任何通过 PSK 的身份能订阅任意 room**。ACL 限的是"哪些设备能碰 broker"，**限不了一个 token 能进哪些 room**（per-room 授权是 §11.2 演进档）。别误以为 ACL/PSK 提供了房间级隔离。

---

## 3. 网络层：Tailscale 装机入网

系统级安装 tailscaled（内核模式），各机登录**同一 tailnet**：

- broker 机：`tailscale up --advertise-tags=tag:broker`
- 参与者机（跑 Claude/Codex）：`tailscale up --advertise-tags=tag:agent`
- 无人值守机器（常开 broker / 算力机）：用 **ephemeral + tagged auth key**；泄漏即在控制台 **Revoke**（网络层可吊销，应用层 PSK 目前不行——见 §6）。

## 4. 网络层：套用 ACL 模板

把 [`examples/tailscale-acl.hujson`](../examples/tailscale-acl.hujson) 粘进 Tailscale 管理台 Access Controls：

- 🔴 **先删默认 allow-all**（新 tailnet 默认全通；不删则 ACL 形同虚设）。
- 🔴 **端口 = broker 实际端口 4700**（`DEFAULT_BROKER_PORT`）。曾经规格附录写成 4500，会让 4700 仍被默认 deny 挡死。

---

## 5. 启动 broker（绑 100.x，**不绑 0.0.0.0**）

```bash
tailscale ip -4                    # 取本机 100.x 地址
abg broker start --host 100.x.y.z --port 4700
```

- 解读启动告警：`resolveBindHost` 对 loopback / Tailscale CGNAT（100.64.0.0/10）静默放行；**看到"绑定非 loopback 地址会暴露给物理 LAN"告警 = 你绑错了**（`0.0.0.0` / `::` / LAN IP / 公网 100.x 都会告警）。空串会被当作 loopback `127.0.0.1` 静默回退（有意的安全默认），不告警。
- 🔴 **绝不 `--host 0.0.0.0`**：那会同时把 broker 暴露给物理 LAN/WiFi。
- 传输层：Tailscale 形态用明文 `ws://`，靠 WireGuard 加密，**不需要也没有内置 TLS**；公网形态必须前置反代终结 `wss://`。
- 优雅停机：`SIGTERM`/`Ctrl-C` 会停服 + checkpoint WAL（pending 落盘后再退），可放心交给 systemd（见 §11）。

---

## 6. 应用层：签发 + 分发 PSK

```bash
abg auth login --id <email|github> --name <显示名>   # → 打印 token
```

- token 落 `<state>/auth-token`（0600）+ `collab.db`（0700 目录）；broker 从不回显。
- **带外分发**（IM / 密码器），勿提交 git。
- ⚠️ **目前没有 revoke/rotation CLI**：泄漏的 PSK 永久有效。临时手段=换 token 重签 + 清理旧库（token hashing at rest / revoke 列入 backlog）。

---

## 7. 参与者连入远程 broker

```bash
export AGENTBRIDGE_BROKER_URL=ws://100.x:4700/ws        # 或 MagicDNS：ws://<host>.<tailnet>.ts.net:4700/ws
```

- `resolveBrokerUrl` 优先级：显式 > `AGENTBRIDGE_BROKER_URL` > 本地默认 `ws://127.0.0.1:4700/ws`。
- 说明：规格里 `abg join <host> --room <码>` 是**未来语法糖**；**今天选远程 broker 走 env**，`abg join <roomId>` 仍是本地 room 操作（写本地 collab DB + cwd→room 映射）。
- 长连韧性：首连可能走 DERP 中继再升级直连；BrokerClient 自带指数退避 + jitter 重连（§8.2），broker blip / 重启会自动恢复。

---

## 8. 数据面：共享 git remote（前提，非 AgentBridge 能力）

控制面只传**事件**，代码同步靠 **git**（§2.6）。完成事件里的 `repo/branch/commit` 是 git **指针**，broker 永不传文件。运维步骤：

1. 约定同一 git remote（GitHub / 内网 Gitea）。
2. 各参与者各自 `clone` / worktree。
3. 被派方收到完成事件后，先 `git fetch` 拿到对应 commit 再起 session。
4. 例行 push/pull 同步。

## 9. 跨机验收清单（镜像 §13，可勾选）

- [ ] broker 经 `100.x` 内网可达（`curl http://100.x:4700/healthz` 返回 `{ok:true,...}`）
- [ ] 无 `tag:agent` 的设备连不上（删一台设备的 tag 实测被拒）
- [ ] 一端完成 → 同 room 成员下一步自动获知（含摘要 / 仓 / 契约）
- [ ] DM 只到目标，不打扰房间其他 session
- [ ] 新成员 join → 拿到白板摘要
- [ ] 离线成员重连 → 补收事件（含 broker 重启后，pending 跨重启存活）
- [ ] 全程 broker 不传任何代码文件
- [ ] 原 Claude↔Codex 单机流不受影响

> 上真机前可先用 [`docker/`](../docker/) 多机模拟验证 broker 层（跨容器 PSK + 扇出），再上 tailnet。

## 10. 排障

- 绑了 loopback 却收不到 `tailscale0` 进来的连接 → 用 `tailscale ip -4` 的 100.x 重绑。
- ACL 没生效 → 多半默认 allow-all 没删。
- 健康探针：`GET /healthz` 返回 `{ok,pid,uptimeMs,connections}`（无 token/PII），可作 watchdog 探活。
- 容器里 tailscale 用 userspace-networking 时记得配 SOCKS5。
- 首连慢 → 等 DERP 透明升级直连。

## 11. 常开与自动拉起（watchdog）

MVP 推荐交给 **systemd**（broker 形态本就是常开机）：

```ini
# /etc/systemd/system/agentbridge-broker.service（示意）
[Service]
ExecStart=/usr/local/bin/abg broker start --host 100.x.y.z --port 4700
Restart=always
RestartSec=2
```

`abg broker start` 已实现优雅关闭（SIGTERM → 停服 + WAL checkpoint + exit 0），配 `Restart=always` 即得"挂了自动拉起"。自建 `abg broker watch` 监督进程列入 backlog（systemd 不够用时再做）。
